Q1. Differentiate arrow functions and regular functions.

Arrow Functions Regular Functions
The first, usual way, is by using the function keyword arrow functions do not have their own arguments and it uses the arguments from the outer function
A regular function, execution context (i.e. the value of this) is dynamic. This means that the value of this depends on how the function was invoked (simple invocation, method invocation, indirect invocation or constructor invocation). an arrow function does not define its own execution context. This results in an arrow function's this being resolved lexically (i.e. the scope in which the arrow function was defined).
Regular functions can be used as constructors, using the new keyword. another consequence of the lexical resolution of this inside arrow functions is that they cannot be used as constructors.
function MyFunction() { console.log("this"); } const Car = (color) => { console.log("this"); };
